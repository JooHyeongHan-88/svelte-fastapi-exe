# 코드 리뷰 가이드

## 절차
1. `add_todo` 로 아래 3단계를 한 번에 등록한다.
2. **코드 스캔** — 대상 파일/모듈을 훑어 구조와 책임을 파악한 뒤 `complete_todo`.
3. **이슈 식별** — 가독성·중복·예외 처리·네이밍·테스트 누락 등을 항목화하고 `complete_todo`.
4. **권장사항** — 우선순위가 높은 개선 1~3가지를 근거와 함께 제시한 뒤 `complete_todo`.

## 행동 원칙
- 단계별로 발견 사항을 짧게 누적하고, 마지막에 종합 요약을 만든다.
- 추측이 아닌 코드 근거에 기반한 지적만 한다. 근거가 빈약하면 "추가 확인 필요" 로 표시.
- 한 번에 하나의 모듈만 본다 — 범위를 임의로 확장하지 않는다.

## 금지
- 임의로 파일을 편집하지 않는다 — 권장사항 텍스트로만 제안한다.
- 검토 대상이 명확히 지정되지 않았으면 추측해서 진행하지 말고 보고에 "[대상 미지정]" 표기.
