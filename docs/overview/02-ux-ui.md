# ② 구현된 UX/UI

> **대상 독자**: 이 앱의 사용자 경험을 처음 파악하는 사람
> **이 문서의 목표**: 최종 사용자가 만나는 화면 구성과 기능을, 실제 사용 흐름 순서대로 이해한다.

---

## 1. 화면 전체 레이아웃

3단 구성: **사이드바(세션) — 채팅 영역 — 아티팩트 패널(산출물)**.

```
┌──────────┬──────────────────────────────┬───────────────┐
│          │  TopBar  (현재 세션 제목)       │               │
│ Sidebar  ├──────────────────────────────┤  Artifact     │
│          │                              │  Panel        │
│ + 새 대화 │  ChatArea                    │               │
│          │   ┌ user 말풍선               │  ┌ 칩 탭 바    │
│ 세션 목록 │   └ assistant 말풍선          │  └ 활성 칩     │
│ (오늘/    │      ├ 스킬 뱃지              │     콘텐츠     │
│  어제/    │      ├ 작업 타임라인(세그먼트)  │   · 이미지     │
│  이번 주) │      ├ TodoProgress 체크리스트│   · 차트      │
│          │      ├ 본문 (마크다운)         │   · 마크다운   │
│ ──────── │      └ AskUserCard (필요 시)  │   · 데이터     │
│ Model    ├──────────────────────────────┤               │
│ Picker   │  Composer  (입력창)           │ ←드래그 리사이즈│
│ 설정·테마 │  Enter 전송 · / 스킬 호출      │               │
└──────────┴──────────────────────────────┴───────────────┘
```

| 영역 | 컴포넌트 | 역할 |
|---|---|---|
| 좌측 | `Sidebar` `SessionItem` `ModelPicker` | 세션 목록·새 대화·모델 전환·설정·테마 |
| 중앙 상단 | `TopBar` | 세션 제목, 모바일에서 사이드바 토글 |
| 중앙 | `ChatArea` `MessageBubble` | 대화 스크롤 영역 (하단 자동 스크롤) |
| 중앙 하단 | `Composer` | 입력창 (auto-resize, 슬래시 커맨드) |
| 우측 | `ArtifactPanel` + 4종 뷰어 | 에이전트 산출물 표시 (탭형 칩) |

## 2. 세션(대화) 관리 — Sidebar

여러 개의 대화를 동시에 보관하고 전환할 수 있다. **모든 세션·메시지는 브라우저
localStorage에 저장**되므로 EXE를 껐다 켜도, 탭을 닫았다 열어도 대화가 유지된다.

| 기능 | 사용 방법 | 비고 |
|---|---|---|
| 새 대화 | `+ 새 대화` 버튼 | 빈 세션 생성 후 즉시 전환 |
| 세션 전환 | 세션 행 클릭 | 백엔드 LLM 컨텍스트도 함께 복원(hydrate) |
| 이름 변경 | 세션 행 **더블클릭** → 인라인 편집 | 수동 변경 후엔 자동 제목이 덮어쓰지 않음 |
| 삭제 | 행 hover 시 나타나는 삭제 버튼 | 백엔드 히스토리도 함께 삭제 |
| 자동 제목 | 첫 메시지 입력 시 자동 생성 | `autoTitle()` — 첫 사용자 문장 기반 |
| 시간 그룹핑 | 자동 | "오늘 / 어제 / 이번 주 / 이전" 버킷으로 정렬 표시 |

안전장치: **응답 생성 중(`streaming`)에는 세션 전환·삭제·새 대화가 모두 차단**된다 —
진행 중인 턴이 다른 세션에 섞여 저장되는 사고를 원천 방지.

모든 세션을 삭제해도 서버는 꺼지지 않는다 (브라우저가 열려 있는 한 keepalive 채널 유지).

## 3. 메시지 입력 — Composer

| 기능 | 동작 |
|---|---|
| 전송 | `Enter` |
| 줄바꿈 | `Shift + Enter` |
| 입력창 크기 | 내용에 따라 자동 확장 (auto-resize) |
| 생성 중단 | 생성 중 정지 버튼 / `ESC` — 받은 데까지 표시되고 "중단됨" 상태로 저장 |
| 생성 중 입력 | 비활성화 (이중 전송 가드) |

### 슬래시 커맨드 — 스킬 강제 활성화

`/` 를 입력하면 등록된 SKILL 목록이 자동완성 패널(`SkillPicker`)로 뜬다.
선택한 스킬은 키워드 트리거 매칭을 우회하고 **그 턴에 반드시 활성화**된다.

```
/data_summary  ← 입력 시 "데이터 요약" 스킬이 무조건 적용된 상태로 전송
```

### 메시지 되감기 (rewind)

지난 **user 메시지에서 되감기를 실행하면** 그 시점 이후의 대화가 잘려나가고, 원래
입력했던 내용이 Composer에 다시 채워진다 → 질문을 고쳐서 다시 보내는 UX.
백엔드 LLM 컨텍스트도 잘린 시점으로 함께 복원된다.

## 4. 응답 표시 — MessageBubble

| 역할 | 렌더링 |
|---|---|
| user 메시지 | plain text (`white-space: pre-wrap`) |
| assistant 메시지 | **마크다운 렌더링** — marked + DOMPurify(XSS 방어) + highlight.js(코드 하이라이트) |

응답은 SSE(Server-Sent Events)로 **토큰 단위 실시간 스트리밍**된다. 완료된 메시지에는:

- **완료 마커** (말풍선 상단 회색 점) — ESC로 중단된 메시지에는 표시 안 함
- hover 시 **소요 시간** footer (`12.3초` 형식)

## 5. 작업 진행의 실시간 가시화

단순 채팅과 차별화되는 핵심 UX: **에이전트가 지금 무엇을 하고 있는지 항상 보인다.**

### TurnStatus — "지금 뭐 하는 중인가"

생성 중인 말풍선 하단에 펄스 점 + 상황별 문구 + 경과 시간이 표시된다.
문구는 진행 상태를 검사해 자동 선택된다:

| 상황 | 표시 문구 |
|---|---|
| 도구 실행 중 | `도구 실행 중…` |
| 서브 에이전트 작업 중 | `에이전트 작업 중…` |
| 추론(reasoning) 직후 | `추론 중…` |
| 그 외 | `응답 생성 중…` |

### TodoProgress — 작업 계획 체크리스트

에이전트가 다단계 작업을 시작하면 `add_todo`로 계획을 등록하고, UI에 체크리스트가
실시간 갱신된다. 각 항목은 대기 → 완료/실패/건너뜀 상태와 한 줄 결과 요약을 보여준다.

```
작업 진행                       ┐
 ✓ 데이터 조회 — 1,234행 조회 완료 │ 단계가 끝날 때마다
 ✓ 통계 산출 — 평균·분산 계산      │ 실시간으로 체크됨
 ◌ 차트 생성                     ┘
```

### 스킬 뱃지

- **SkillBadge**: 턴 시작 즉시 활성화된 SKILL 이름 표시 (어떤 작업 지침이 적용됐는지)
- **SkillCompleteBadge**: 계획된 todo가 전부 종결되면 "작업 완료" 뱃지 표시

## 6. 에이전트 활동 타임라인 — 세그먼트

assistant 말풍선 안에는 본문 텍스트 외에 **활동 세그먼트**가 시간 순으로 쌓인다.

| 세그먼트 | 컴포넌트 | 표시 내용 |
|---|---|---|
| 추론 | `ReasoningBlock` | LLM의 중간 판단 설명 (접이식 블록) |
| 도구 실행 | `ToolStep` | `🔧 도구명 호출 중…` → 완료/에러 상태 카드 |
| 서브 에이전트 | `SubAgentStep` | 위임 트레일 — 아래 상세 |

### 서브 에이전트 트레일

오케스트레이터가 전문 에이전트에게 작업을 위임하면 **트레일 카드**가 생긴다.
카드 내부에 그 에이전트의 도구 실행·todo가 실시간으로 채워지고, 완료 시 요약과 함께 닫힌다.

```
🔄 orchestrator → analyst_agent   (진행 중)
    ├ 🔧 exec_code 실행 중…
    └ ✓ 통계 산출 완료
✓ orchestrator → analyst_agent   "분석 완료. 평균 4.2, 이상치 3건 발견."
```

**병렬 위임**(`call_sub_agents_parallel`) 시에는 트레일 여러 개가 **동시에 진행**되는
모습이 인터리브되어 표시된다 — 같은 이름의 에이전트 2개가 동시에 떠도 `dispatch_id`
상관키로 각자의 트레일에 정확히 라우팅된다.

## 7. 사용자 되묻기 — AskUserCard

에이전트가 진행에 필요한 정보가 부족하면 **카드 형태로 질문**하고 턴을 멈춘다.

| input_type | 화면 | 답변 방법 |
|---|---|---|
| `choice` | 선택지 버튼만 | 버튼 클릭 |
| `text` | 질문 텍스트만 | Composer에 직접 입력 |
| `both` | 버튼 + 직접 입력 힌트 | 둘 다 가능 |

```
┌────────────────────────────────────┐
│ 어느 기간의 보고서를 생성할까요?        │
│ [오늘] [이번 주] [이번 달] [직접 입력] │
└────────────────────────────────────┘
```

발동 경로는 두 가지다 — 사용자 입장에선 동일한 카드로 보인다:

1. **LLM의 능동 질문** (`ask_user` 도구) — 요청이 모호할 때
2. **슬롯 가드 자동 발동** — 도구 실행에 필수 인자가 빠졌을 때 (답변하면 자동 재시도)

## 8. 아티팩트 패널 — 산출물 칩 4종

에이전트가 만든 산출물(이미지·차트·보고서·데이터)은 채팅에 섞이지 않고 **우측 패널**에
표시된다. 메시지 말풍선에는 **칩**이 남아, 클릭하면 언제든 패널이 다시 열린다 —
세션을 나갔다 돌아와도 칩과 산출물 파일이 보존되므로 재열람 가능.

| 칩 | 패널 뷰어 | 내용 |
|---|---|---|
| 🖼️ 이미지 N장 | `ArtifactImage` | 세로 갤러리 — 6장씩 lazy load(IntersectionObserver) 무한 스크롤, 클릭 시 라이트박스 |
| 📊 차트 N개 | `ArtifactChart` | ECharts 반응형 그리드 — 페이지당 최대 6개 + 페이지네이션 |
| 📝 문서 제목 | `ArtifactMarkdown` | 마크다운 보고서 렌더링 (채팅과 동일한 sanitize 파이프라인) |
| 데이터 (parquet) | `ArtifactData` | head(10) 미리보기 테이블 + 전체 CSV 다운로드 |

패널 공통 UX:

- **드래그 리사이즈** — 경계를 끌어 너비 조절 (최대 1000px, viewport 60% 캡), 설정은 localStorage에 기억
- display 계열 칩은 생성 즉시 패널 자동 오픈
- **데이터 칩은 자동 오픈하지 않음** — 전처리 중간 저장이 빈번해 패널이 튀는 것을 방지

### 데이터 칩 상세 (parquet 중간 산출물)

에이전트가 분석 중간 데이터를 parquet으로 저장할 때마다 칩이 생긴다.

- 패널에서 **상위 10행 미리보기** (가로 스크롤·고정 헤더·컬럼 타입 표기) + 행×열·크기·경로 메타
- **CSV 다운로드** — 저장 위치 선택 다이얼로그(`showSaveFilePicker`), 미지원 브라우저는 일반 다운로드 폴백

## 9. 차트 인터랙션 — 라이트박스에서 탐색·필터·편집

차트 셀을 클릭하면 **전체화면 라이트박스**(`ArtifactLightbox`)가 열린다. 단순 보기를
넘어 **데이터 탐색 도구**로 동작한다.

| 조작 | 방법 |
|---|---|
| 확대 보기 | 차트 셀 클릭 → 라이트박스, 우하단 핸들 드래그로 크기 조절 |
| 차트 간 이동 | ← / → 방향키 |
| 영역 선택 | brush 드래그 (scatter·line·ecdf 등 행 단위 차트) |
| 행 제외 필터 | 선택 후 **Filter**(이 차트만) / **Filter All**(같은 데이터를 쓰는 모든 차트) |
| 그룹 제외 | 레전드 패널에서 그룹 체크 → 같은 Filter 버튼 |
| 실행 취소 | **Undo / Redo / Reset** — 필터·레전드 변경을 하나의 히스토리로 되감기 |

### 레전드 편집 패널

color 그룹이 2개 이상인 차트에서 **Legend** 버튼을 누르면 우측에 편집 패널이 열린다.

```
┌ Legend ──────────────┐
│ ⠿ ● 그룹A   👁  ☐    │   ⠿ 드래그 = 순서 변경
│ ⠿ ● 그룹B   👁  ☑    │   ● 클릭 = 색상 변경 (컬러피커)
│ ⠿ ● 그룹C   🚫 ☐    │   👁 = 표시/숨김 토글
└──────────────────────┘   ☑ = Filter 대상 선택
```

핵심 설계: 그리드와 라이트박스가 **같은 캐시를 공유**하므로, 어느 쪽에서 필터·편집해도
양쪽이 동시에 갱신된다. 필터 상태는 서버 파일로 영속되어 **세션을 나갔다 와도 유지**되고
Undo로 언제든 복원 가능하다.

## 10. 산출물 재인용 — `@` 참조 삽입

칩(말풍선)과 패널 헤더에 있는 **`@` 버튼**을 누르면 해당 산출물의 경로가 입력창에
삽입된다 — *"이 데이터로 이어서 분석해줘"* 류의 후속 요청을 클릭 한 번으로 지정하는 UX.

```
[데이터 칩 @ 클릭]
Composer: "result/세션명-a1b2c3d4/20260611-103000/data.parquet 이걸로 추세 분석해줘"
```

백엔드의 산출물 재발견 도구(`list_artifacts`/`load_artifact`)가 이 경로를 그대로
해석하므로, 과거 턴·과거 세션의 산출물도 이어서 작업할 수 있다.

## 11. LLM 설정 — SettingsModal & ModelPicker

### SettingsModal (사이드바 ⚙ 아이콘)

| 항목 | 설명 |
|---|---|
| 프로바이더 선택 | `mock` / `dtgpt` / `openai_compatible` |
| 모델·API Key·Base URL | 프로바이더별로 입력 (DTGPT는 Base URL 필드 숨김 — 환경 변수 고정) |
| 연결 테스트 | 저장하지 않고 임시 설정으로 ping |
| 멀티 프로바이더 캐시 | 프로바이더를 전환해도 이전 프로바이더의 설정이 보존됨 |

보안: API 키는 **백엔드 settings.json에만 저장**되며 브라우저에는 절대 저장하지 않는다.
조회 시 항상 마스킹(`sk-p••••••4f2a`)되고, 입력 필드는 placeholder로만 흔적을 보여준다.

### ModelPicker (사이드바 하단)

현재 `프로바이더 / 모델`을 상시 표시하고, 클릭하면 **드롭업**으로 모델을 빠르게 전환한다.

- 모델 목록은 프로바이더의 `/models` 엔드포인트에서 로드 (5분 캐시)
- 목록이 5개를 넘으면 검색창 자동 표시
- 선택 즉시 저장 — **서버 재시작 없이 다음 메시지부터 적용** (provider hot-swap)

---

## 12. 자동 업데이트 UX

| 단계 | 화면 |
|---|---|
| 새 버전 감지 | 상단에 `UpdateBanner` 표시 — "새 버전 X.X.X 사용 가능" |
| 업데이트 실행 | 배너 클릭 → `UpdateModal` — 다운로드 진행률 바 → 검증 → 재시작 안내 |
| 교체·재기동 | 앱이 스스로 종료 → Updater가 EXE 교체 → 새 버전 자동 실행 |

사용자가 하는 일은 배너 클릭 한 번. 실패 시 자동 롤백되어 기존 버전이 다시 뜬다.

## 13. 테마·반응형

- **라이트/다크 테마** — 사이드바 토글, 선택은 localStorage에 기억
- 코드 하이라이트·차트·모든 컴포넌트가 CSS 변수 토큰 기반이라 테마 전환 즉시 반영
- 좁은 화면(모바일)에서는 사이드바가 TopBar 버튼으로 접힘
