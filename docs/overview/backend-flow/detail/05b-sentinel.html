call_sub_agent / call_sub_agents_parallel

call_sub_agent(agent_name, task) → 순차 디스패치
→ 06 dispatch _dispatch_sub_agent
오케스트레이터만 가능(agent_registry is not None). 서브가 재호출하면 L0 차단.
call_sub_agents_parallel(tasks=[...]) → 병렬 디스패치
→ 06 dispatch _dispatch_parallel_sub_agents
독립 작업 N개를 semaphore(APP_MAX_PARALLEL_SUBAGENTS) 안에서 동시 실행. 단일 tool_result 로 fan-in.
8

complete_subagent — 서브에이전트 종료

complete_subagent(summary) 호출 (서브에이전트 전용)
harness/call_handlers.py · sentinel=True
tools 화이트리스트에 명시하지 않아도 harness 가 항상 자동 주입.
ToolResultEvent(content=summary) yield
summary 는 AgentReturnEvent.summary 로 캡처. 우선순위: complete_subagent → _extract_task_summary("Task Summary:" 헤더) → 마지막 200자 폴백.
outcome.stop = True → _run_agent_turn 즉시 종료
sentinel vs 일반 도구: @register_tool(sentinel=True) 로 등록된 도구는 _execute_tool 에 도달하면 명시적 에러(L3). 하니스의 분기 처리가 먼저 가로채야 하므로, _SENTINEL_ROUTES 의 조건이 뚫려야만 L3 에 닿는다.
