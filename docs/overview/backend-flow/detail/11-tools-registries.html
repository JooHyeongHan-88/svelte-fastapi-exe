Tools & Registries

⑪ 도구 등록·검증·실행 + 4 레지스트리

"파일 하나 = 도구 하나" 자동 등록과, 마크다운/데코레이터 정의를 런타임 객체로 공급하는 4개 레지스트리. 메타는 부팅 1회, 본문은 lazy.

1

도구 생애 — 등록 → 검증 → 실행

등록
@register_tool — 부팅 시 자기등록
registries/tools.py · register_tool() · ToolRegistry · RegisteredTool · registry
import agent.tools 부수효과로 _REGISTRY 채움. 시그니처의 Annotated[T, "설명"] → Pydantic create_model 로 JSON Schema(LLM 노출) + TypeAdapter(검증) 1회 생성.
검증
guard.validate_tool_args() → _split_errors()
guard.py · MissingSlot · SlotCheckResult
◇ ValidationError 종류?책임자별 분기 — 사용자 책임 vs LLM 책임
missing — 값 부재
AskUserEvent → 사용자에게 재질문
slot_prompts 에서 질문 문구 · outcome.interrupted=True
invalid — 형식 오류
ToolResultEvent(is_error) → LLM self-correct
사용자 미개입. 동시에 missing 도 있으면 이 경로가 포함 안내.
통과
실행 — 일등시민 타임아웃
tool_exec._execute_tool()
asyncio.wait_for(timeout=tool.timeout_seconds) · 기본 APP_TOOL_DEFAULT_TIMEOUT(30s)
timeout → [timeout], 예외 → [error type]. sentinel 이 흘러오면 명시적 에러(L3).
Sentinel 도구: add_todo 처럼 하니스가 직접 가로채는 도구는 sentinel=True 로 등록 — 함수 본문 미실행, 호출 신호 역할만.

4개의 Registry — "정의 파일 → 런타임 객체"

Registry원천관리 대상 / 로딩 정책
PromptRegistryPROMPTS/*.md기반 system prompt 합성 compose() — 매 턴 읽기, dev mtime 핫리로드
SkillRegistrySKILLS/*.md트리거 매칭 select() / 슬래시 get_by_names() — 메타 부팅 1회, 본문 lazy
AgentRegistryAGENTS/*.md서브에이전트 카탈로그, Case 3 매핑 — 메타 부팅 1회, 본문 위임 시 lazy
ToolRegistry@register_tool스키마·검증기·실행 함수 — import 시 자기등록(부팅 1회)
공통 패턴: 메타데이터(Front Matter)는 가볍게 미리, 본문은 필요할 때 lazy — 파일이 늘어나도 부팅 비용이 늘지 않는다. frozen 은 부팅 시 content_sync 가 성공하면 use_directory() 로 동기화본을 읽는다.
4

오케스트레이터 라우팅 — Case 0~5

Case조건 → 행동
0요청 모호 → ask_user 로 먼저 질문
1일상 대화 → 도구 없이 즉시 텍스트
2에이전트 지명 → 즉시 위임
3SKILL 트리거 + 전담 에이전트 존재 → 자동 위임(직접 실행 금지)
4SKILL 매칭 but 전담 에이전트 없음 → 직접 도구 실행
5마지막 응답 → "무엇을/결과는/다음은" 완료 보고
Case 3 가 핵심: SKILL(무엇을) × AGENT(누가)의 결정론 매핑 — LLM 임의 판단이 아니라 설정 파일이 라우팅을 결정.
