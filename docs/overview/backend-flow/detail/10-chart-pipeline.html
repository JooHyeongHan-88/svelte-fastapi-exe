Chart Pipeline

⑩ 차트 파이프라인 — 선언적 spec & 인터랙션

display_chart 는 "이미지 생성"이 아니라 데이터(parquet)와 선언(spec)을 분리해 렌더링하는 파이프라인. 덕분에 사후 필터링·레전드 편집(undo/redo)이 가능하다.

1

생성 → 상호작용 흐름

생성 (에이전트)

  1. 데이터 저장save_artifact(parquet)
  2. 차트 선언charts.spec.json (ChartSpecV1)
  3. 표시 호출tools/visualize.py · display_chart(spec 경로)
  4. 렌더chart_renderer.py · render_spec_to_echarts() → charts.json

상호작용 (사용자)

  1. brush/레전드 선택ArtifactLightbox
  2. 필터 요청POST /api/chart/filter
  3. ViewState pushchart_filter_store.py (undo/redo)
  4. 재렌더제외 행·레전드 반영 → charts.json 갱신
2

선언 스키마 — chart_spec.py

클래스역할
ChartSpecV1최상위 — charts 목록 선언 (Vega-Lite 류 축소판)
ChartV1mark(bar/line/scatter/box/histogram/heatmap/ecdf) + encoding + data
Encoding / EncodingChannelx / y / color 채널 (field·type·aggregate)
DataRefdata.source — parquet 경로 (resolve_result_path 로 해석)
관용 파싱 (R8): _normalize_alias field_validator(mode="before")가 근사 표기를 정규화(normal→nominal, avg→mean, hist→histogram, 대소문자 무관). 매핑 밖 값만 ValidationError. display_chart 오류 회신은 최대 3건(_SPEC_ERROR_DETAIL_LIMIT).
3

렌더러 — chart_renderer.py

render_spec_to_echarts(spec, base_dir, exclude_by_chart, legend_by_chart)
_load_data_frame → _render_chart (mark별 분기) → _apply_legend_config
parquet 로드 → mark별 빌더(_render_standard/_render_box/_render_histogram/_render_heatmap/_render_ecdf) → 레전드 설정 적용.
▽ R8 관용 보정 — mark 별로 병렬 처리
R8 — histogram + color
_render_histogram
그룹별 시리즈 분리
빈 경계는 전체 범위에서 1회 계산해 그룹 공유(비교 가능).
R8 — 레전드 라벨 보정
_restore_mismatched_legend
extra_option.legend.data 임의 라벨
실존 시리즈 이름과 교집합 유지, 전부 어긋나면 시리즈 이름으로 복원.
레전드 Filter 환원
resolve_legend_row_ids
color.field 컬럼 기반
레전드 이름 → 행 역추적 → exclude 환원.
∥ → charts.json (ECharts option 직렬화)
4

ViewState 스택 — chart_filter_store.py

v2 스키마로 필터(exclude)와 레전드 오버라이드(legend)를 단일 undo/redo 스택에 통합. cursor 가 가리키는 ViewSnapshot 이 현재 상태.

전이 함수 / action효과
apply_exclude(state, ...)brush/레전드 Filter → 행 제외 push (legend carry)
apply_legend(state, ...)순서·색상·Hide → legend 갱신 push (exclude carry)
reset / undo / redo빈 스냅샷 push / cursor 이동(양쪽 동작 무관하게 되감음)
/api/chart/filter (api/chart.py) action: exclude · exclude_legend · set_legend · undo/redo/reset. 응답 {items, can_undo, can_redo} 로 charts.json 도 덮어 재진입 일관성 보장. 데이터 원본(parquet)은 불변 — 필터는 row id 제외 목록.
5

데이터 칩 HTTP — api/artifact.py

엔드포인트동작
GET /api/artifact/previewhead(N) 미리보기 — scan_parquet 로 total_rows 집계 + read_parquet(n_rows). NaN/inf→null
GET /api/artifact/csv전체 데이터 CSV 변환 첨부(RFC 5987 한글 파일명)
POST /api/artifact/reveal산출물 폴더를 OS 탐색기로 열기(Windows os.startfile)
