System Prompt Composition

⑦ system prompt 조립

run_turn 이 매 턴 system prompt 를 동적으로 합성하는 서브패키지. 오케스트레이터 / 서브에이전트 / 단층 세 경로를 공통 섹션 렌더러로 조립한다.

0

모듈 구성

모듈역할
compose.py세 경로 최종 조립 (_compose_orchestrator / _compose_sub_agent / _compose_system_prompt)
sections.py공통 섹션 렌더러 (비활성 카탈로그·멀티스킬·To-do·Pending) — 모두 str | None 순수 함수
artifacts.py_render_session_artifacts_section — 세션 manifest 기반 과거 산출물 목록
api_refs.py_render_skills_api_refs · _collect_agent_api_refs_section — api_refs → ApiDoc 섹션
wind_down.py_build_wind_down_message — 반복 예산 임박 마무리 지시문 (R7)
base prompt(PROMPTS/base.md + safety.md (+ orchestrator.md))는 PromptRegistry.compose() 가 만든다. 이 서브패키지는 그 base 뒤에 런타임 상태(SKILL 본문·todo·산출물·카탈로그)를 이어 붙인다.

세 경로 비교

오케스트레이터

_compose_orchestrator_system_prompt

AGENTS 보유 시. 카탈로그 + Case 3 결정론 매핑("X 트리거 → 반드시 agent 위임")으로 설정이 라우팅을 결정.

서브에이전트

_compose_sub_agent_system_prompt

base(orchestrator 제외) + 페르소나 본문 + 학습 SKILL + api_refs + 종료 규약(complete_subagent 필수). call_sub_agent 도구는 시야에서 제거.

단층 (하위호환)

_compose_system_prompt

AGENTS 부재 시. 카탈로그·Case 3 없이 base + SKILL + todo + 산출물 + Pending.
