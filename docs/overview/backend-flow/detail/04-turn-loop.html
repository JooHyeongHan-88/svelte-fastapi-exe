영속화 & 실패 턴 복구 (R1)

정상 영속화
conversation_store.append(F7) + state_store.set + clear_all_pending(F11)
tool 결과 800자 절단(F7) · pending 클리어(AskUser 없이 완료됐으면)
F11: AskUserEvent 가 한 번도 yield 되지 않았으면 pending_tool·pending_args·missing_slots·pending_sub_agent 전부 클리어. stale pending 이 다음 턴 system prompt 오염 방지.
■ DoneEvent 정확히 1회 — 정상 종료
예외 경로 (R1)
run_turn() 최상위 except Exception
harness/loop.py · turn_persisted 플래그로 재-append 방지
순서: ① _balance_all_unresolved(turn_messages) — 미해결 tool_call placeholder 전수 삽입. ② F11 pending 클리어 → store.append + state_store.set. ③ ErrorEvent(F12 안전화 — str(exc) 대신 타입명만). ④ DoneEvent 정확히 1회. 영속 실패 시: 로그만 남기고 ErrorEvent/DoneEvent 는 막지 않음. CancelledError(ESC)는 BaseException: 이 경로를 타지 않음(중단 턴은 백엔드 미영속 = 의도된 동작). 그 결과 생기는 프론트 localStorage ↔ 백엔드 store desync 는 프론트가 치유한다 — stopStreaming 이 플래그를 세우고 다음 sendMessage 가 chat() 직전 restoreConversation 으로 정제본을 재주입(→ frontend_state.md "ESC 재동기화").
