tool_call Handling

⑤ tool_call 핸들러 — 3단계 결정트리

_run_agent_turn 이 emit한 tool_call 하나마다 _handle_tool_call(ctx, call, outcome) 가 처리한다. 깨진 인자 게이트 → sentinel 게이트 → 일반도구 파이프라인의 결정트리로 표현한다.

1

데이터클래스

TurnContext

한 턴 전체에서 고정된 실행 환경. 핸들러 시그니처를 (ctx, call, outcome) 로 통일. 가변 컬렉션(messages·history_calls·active_skills)은 in-place 갱신. is_sub_agent = turn_messages is None.

CallOutcome

tool_call 처리 후 호출부가 읽는 제어 신호. 둘 다 False 면 CONTINUE.

interrupted — ask_user·슬롯 누락 → 루프 break.
stop — complete_subagent → 즉시 종료.
