Harness Resilience

⑫ 안전장치 카탈로그 (F1~F12 · R1~R8)

실 LLM 의 비결정성·실패 경로를 흡수하는 방어선들. Mock 시나리오는 happy path 만 검증하므로, 이 불변식들이 실 LLM 투입 시 임계적이다. "이 방어 코드가 왜 있나, 회귀 금지"의 카탈로그.

1

분류별 요약

분류장치효과
입력 경계Origin 가드 + 루프백 고정외부 네트워크에서 API 접근 불가
입력 경계동시 턴 가드 R2같은 세션 중복 요청 → 히스토리 교차 오염 방지
LLM 오류슬롯 가드 책임자 분기사용자 질문 최소화, LLM 실수는 자가수정 유도
LLM 오류깨진 tool_call JSON 마커 F3스트림 잘림 시에도 원본 보존 → 재요청
폭주 방지반복 상한(12)·턴 예산(20)·loop-guard·wind-down R7무한 루프·비용 폭주 차단
폭주 방지도구 타임아웃(30s)도구 1회가 턴을 영원히 잡지 못함
네트워크provider 재시도(지수 백오프) F4일시 오류(429·연결 끊김) 자동 복구
정합성tool 쌍 보존 트리밍 + placeholder F1aF1bOpenAI 와이어 규약 위반(400) 방지
정합성실패 턴 영속 + DoneEvent 보장 R1예외가 나도 턴이 증발하지 않고 스트림이 닫힘
비밀 보호에러 메시지 안전화 F12 · API 키 마스킹키·URL 이 사용자 화면에 노출되지 않음

F 계열 — 실패 경로 흡수 (Failure)

코드위치내용
F1astores/conversation.pyTool 쌍 보존 트리밍 — 고아 role=tool 이 첫 메시지로 남지 않게 cut 경계 보정
F1bstate/balancing.py배치 중단 시 미해결 tool_call 에 placeholder 채움(_balance_unresolved_tool_calls)
F2providers/openai.py스트림 잘림(finish_reason=length)에도 tool_calls flush + DoneEvent 정확히 1회
F3providers/openai.py · call_handlers깨진 인자 JSON 을 MALFORMED_TOOL_ARGS_KEY 에 원본 보존 → self-correct 요구
F4providers/openai.py_create_stream_with_retry — 일시 오류 지수 백오프(영구 오류는 즉시 전파)
F6harness/loop.py + frontendmax_iterations fallback salvage — todo 전원 terminal=초록(복구), 아니면 빨강(미완)
F7stores/conversation.pyTool 결과 히스토리 800자 절단(현재 턴 컨텍스트는 full 유지)
F11harness/loop.py · state/pending.pyAskUser 없이 끝나면 pending 전부 클리어 — stale pending 오염 방지
F12harness/loop.py · api/chat.py에러 메시지 안전화 — [{type}] 처리 중 오류 (키·URL 미노출)

4

슬롯 가드 — 오류 책임자 분기 (핵심 패턴)

guard.validate_tool_args() → _split_errors()
agent/guard.py
LLM 인자를 도구 Pydantic 모델로 검증하고 오류를 책임자별(누락=사용자 / 형식=LLM)로 분리한다.
◇ ValidationError 종류?
missing — 사용자 책임
MissingSlot → AskUserEvent
slot_prompts 질문 문구. 사용자가 값을 주면 다음 턴 재시도.
invalid — LLM 책임
invalid_message → ToolResultEvent(is_error)
사용자 미개입. 동일한 잘못된 호출 반복은 R4 loop-guard 차단.
주의: save_artifact(kind='json', content=<dict>) 처럼 값은 줬는데 타입이 str 대신 dict 면 형식 오류 → LLM self-correct 경로(사용자 미개입).
