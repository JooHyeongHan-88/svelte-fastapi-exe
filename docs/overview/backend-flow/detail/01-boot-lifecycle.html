Presence & 생명주기 루프

presence SSE 연결 자체가 생존 신호다. 세션 삭제와 서버 생존은 분리돼 있다 — 모든 세션을 지워도 탭이 열려 있으면 서버는 유지된다.

↻ Presence keepalive · 탭이 열려 있는 한 무한 반복 ↻ 30s ping
연결
GET /api/presence (SSE, 연결 유지)
api/presence.py · core/browser.connect_client(id)
세션 있으면 세션 ID, 없으면 BROWSER_KEEPALIVE_ID 로 연결. 30초마다 ": ping" idle timeout 방지.
◇ 탭 닫힘? (EventSource 종료)
아니오 — 살아있음
연결 유지 · ↻ 계속
30s ping → 루프 상단으로
유예
disconnect_client(id) — 2s grace
_pending_disconnects: threading.Timer · PRESENCE_RECONNECT_GRACE
새로고침·탭 복제 흡수. 탭 복제는 _connections 카운트로 — 마지막 연결이 끊겨야 유예 진입.
◇ 2s 후 clients == 0?
아니오 — 다른 탭 있음
타이머 취소 · 계속
Timer.cancel() — lock 밖에서 호출
server.should_exit = True
watchdog → uvicorn 종료
Windows 는 SIGTERM 대신 이 경로만 사용.
동시성: _connections·_pending_disconnects 는 event-loop·watchdog 스레드·Timer 콜백 3곳에서 접근. 모든 read/write 는 _lock 안에서, Timer.cancel/start 는 lock ordering 때문에 _lock 밖에서 호출.
