히스토리 정합성 — balancing.py (F1b · R1)

OpenAI 와이어 규약상 assistant 의 모든 tool_call 에는 매칭되는 role="tool" 메시지가 인접해야 한다. 배치 도중 AskUser·예외로 끊기면 응답 없는 호출이 남아 다음 턴 요청이 400 으로 거부된다.

_balance_unresolved_tool_calls (F1b)

중단(AskUser) 경로 전용. if interrupted: 블록에서 호출. in-flight assistant_msg 의 미해결 tool_call 에 placeholder append.

"[중단됨] 사용자 입력 대기로 …"

_balance_all_unresolved (R1)

예외 경로 전용. in-flight assistant 를 특정할 수 없어 버퍼 전체 순회. 미해결 placeholder 를 해당 tool 블록 바로 뒤(j)에 insert(끝 append 는 인접성 규약 위반).

"[중단됨] 턴이 오류로 종료되어 …"

루프가드 — loop_guard.py (R4)

_call_signature(call) → ( call.name, json.dumps(call.arguments, sort_keys=True), "|".join(sorted(parts)), # result/ 참조 파일 fingerprint "경로:mtime_ns:size" ) # history_calls: set[tuple] 의 원소 타입

파일 fingerprint 를 시그니처에 포함해 "spec 파일을 고쳐 쓴 뒤 같은 경로로 재호출"(정당한 재시도)을 오탐 차단하지 않는다. _record_invalid_call(형식오류 경로)도 같은 history_calls 집합·시그니처를 공유 — 한쪽만 바꾸지 말 것.
