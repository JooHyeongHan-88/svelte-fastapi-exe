Settings & Auto-update

⑬ 설정 hot-swap + 자동 업데이트 / 콘텐츠 동기화

자동 업데이트 — 4단계 결정트리

● GET /api/update/check 요청
◇ QA 채널? (APP_BUILD_CHANNEL==qa)
예 — QA
즉시 차단 — update_available=False
네트워크 호출 없음. prod EXE 가 QA 빌드를 업데이트로 인식하는 것 방지.
■ 종료
① check_latest
GitHub REST API → latest.json 에셋 파싱
updater.py · _fetch_release_and_meta() · 5분 캐시
releases/latest → latest.json(octet-stream) → EXE 에셋 API url stash. url prefix·sha256(64hex) 검증 실패 시 silently False.
② apply_update
PAT + octet-stream 인증 스트리밍 다운로드 → {stem}.new.exe
updater.py · _download() · REST API 에셋 엔드포인트 + Accept:application/octet-stream
검증 통과한 EXE를 PAT 인증 octet-stream으로 스트리밍 다운로드해 {stem}.new.exe로 staging한다.
불일치
staging 파일 삭제 + 에러 반환
■ 중단
◇ sha256 검증 일치?
③ Updater.exe 기동
DETACHED Popen(Updater.exe, pid, new, current) → 1s 후 server.should_exit
부모 pid 폴링 최대 60s + POST_EXIT_GRACE 3s
분리(DETACHED)된 Updater.exe를 띄우고 1초 뒤 자신을 종료해, 교체할 EXE의 파일 잠금을 푼다.
↻ rename 재시도 — 최대 30회 × 0.5s ↻ 0.5s sleep
◇ ④ current→.old rename 성공?AV 스캔·EXE 잠금 해제 대기
실패 — 잠금
↻ 재시도
new→current rename → 새 EXE spawn
notify_shell(SHChangeNotify) → 탐색기 갱신
cleanup_stale_backup() — .old 삭제
■ 업데이트 완료
30회 전수 실패 → .old 복원 후 재기동
current→.old 는 성공했으나 new→current 가 실패한 경우 .old→current 복원.
절대 변경 금지: private 레포 에셋은 브라우저 URL 이 아니라 REST API 에셋 엔드포인트 + Accept: application/octet-stream + PAT 헤더로 받아야 한다. 브라우저 URL 은 쿠키 인증 전용 → private 이면 404.
